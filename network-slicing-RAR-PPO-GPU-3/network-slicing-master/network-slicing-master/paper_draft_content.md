# 面向突发业务的预测增强型RAN网络切片资源动态分配方法

## 摘要

随着5G及后续移动通信网络中业务类型的不断丰富，RAN网络切片需要在有限无线资源条件下同时满足差异化业务的服务等级协议（SLA）要求。尤其在突发流量和资源紧张场景下，传统仅依赖当前观测状态的资源分配方法难以及时响应未来业务波动，容易导致切片SLA违约增加或资源占用偏高。针对上述问题，本文研究一种面向突发业务的预测增强型RAN网络切片动态资源分配方法。该方法利用长短期记忆网络（LSTM）对短时业务需求进行预测，并将预测结果与实时网络状态、跨step资源占用状态及系统剩余资源信息联合输入强化学习智能体，从而学习切片间物理资源块（PRB）的动态分配策略。为更贴近短时资源管理过程，本文在现有切片环境基础上引入跨step资源占用机制与突发业务切换机制，并设计兼顾SLA保障与资源开销的奖励函数。实验结果表明，所构建的预测增强型资源分配框架能够在现有仿真环境中实现稳定接入，并为分析预测信息、持续资源占用机制与切片资源控制行为之间的关系提供实验依据。本文工作可为面向动态业务场景的智能化RAN切片资源管理研究提供参考。

## 1. 引言

随着5G及后续移动通信网络中业务类型的不断丰富，无线接入网需要在有限频谱资源条件下同时支持多种差异化业务。典型业务如增强移动宽带（eMBB）关注高吞吐量与大带宽需求，而大规模机器类通信（mMTC）更强调海量连接与接入稳定性。不同业务在资源需求和服务等级协议（Service Level Agreement, SLA）方面存在显著差异，传统静态或平均化资源分配方式难以在复杂业务环境中同时兼顾服务保障与资源利用效率。网络切片技术通过在共享物理基础设施上构建逻辑隔离的资源实例，为多业务共存提供了有效支撑；然而，如何在RAN切片场景下实现高效、灵活且可持续的动态资源分配，仍是当前研究中的核心问题之一。

针对RAN切片资源分配问题，现有研究主要包括基于优化的方法和基于强化学习的方法。前者通常通过求解静态或逐时隙优化问题为切片分配无线资源，具有较强的建模可解释性，但在业务到达快速变化、网络状态持续波动的场景中往往需要频繁重复求解，难以满足实时控制需求。相比之下，强化学习方法能够通过与环境交互直接学习资源分配策略，在非平稳网络中表现出更强的在线适应能力，因此已被广泛用于网络切片动态资源管理。然而，现有多数强化学习方法仍主要依据当前时刻观测状态做出决策，对未来短时业务波动缺乏显式感知能力，在资源紧张和流量突发条件下容易出现资源配置滞后，进而导致切片SLA违约增加或资源过度占用。

近年来，基于时间序列预测的资源管理方法逐渐受到关注。长短期记忆网络（Long Short-Term Memory, LSTM）能够从历史业务序列中提取时序相关性，对未来短时流量需求进行预测，从而为资源预留和前瞻控制提供信息基础。对于网络切片场景而言，若能将预测结果与强化学习决策过程有效结合，则有望缓解“仅依据当前状态被动分配资源”的问题，提升资源控制的主动性与稳定性。但从现有研究看，一方面，预测信息与资源分配策略之间的耦合方式仍不够清晰；另一方面，许多切片资源分配模型仍将资源动作视为单步即时生效，较少考虑资源分配在相邻决策阶段之间的持续占用效应，从而限制了方法对实际短时资源管理过程的刻画能力。

基于上述分析，本文聚焦固定切片数量场景下的RAN切片动态资源分配问题，研究一种面向突发业务的预测增强型资源分配方法。本文在现有网络切片环境基础上引入突发业务切换机制，使业务负载在正常态与突发态之间动态变化，以构造更具挑战性的资源分配场景；进一步地，利用LSTM对短时外生业务量进行预测，并将预测结果与实时切片状态、跨step资源占用状态及系统剩余PRB信息共同构成强化学习智能体的输入状态，从而增强策略对未来业务波动的感知能力。与此同时，本文将资源动作由单一的即时分配扩展为“跨step占用分配 + 当前step分配”的联合控制形式，使资源策略不仅能够响应当前负载，还能够对后续阶段进行短时资源预留。

在方法实现上，本文围绕eMBB与mMTC切片构建统一的预测增强型资源分配框架。其中，LSTM模块对eMBB业务中的CBR流量、VBR流量以及mMTC业务中的新到达设备数量进行短时预测；强化学习模块则基于增强后的状态信息输出切片间PRB分配策略。为更准确地刻画资源分配目标，本文还设计了兼顾SLA违约惩罚与资源开销约束的奖励机制，将“尽量降低SLA违约风险”与“尽量控制资源占用成本”统一到同一优化目标之下。通过上述设计，本文旨在为突发业务条件下的前瞻性资源控制提供一种可实现的建模与验证框架。

本文的主要工作如下。

1）提出一种融合短时业务预测与强化学习决策的RAN切片动态资源分配框架，通过引入预测信息增强资源分配策略的前瞻性；

2）设计包含跨step PRB占用、未来短时业务预测和系统剩余资源信息的联合状态表示，并构建与之匹配的动态动作机制，以更贴近短时资源持续占用过程；

3）在改造后的网络切片仿真环境中构建突发业务场景与稠密SLA奖励机制，并从SLA违约和资源开销两个维度分析预测增强框架的表现特征。

本文其余部分安排如下：第2节介绍与RAN切片资源分配、强化学习及预测辅助资源管理相关的研究工作；第3节给出系统模型与问题描述；第4节详细介绍所提出的预测增强型动态资源分配方法；第5节给出实验设置与结果分析；第6节总结全文并展望未来工作。

## 2. 相关工作

### 2.1 RAN网络切片资源分配研究现状

随着5G网络业务类型和服务需求的持续丰富，RAN网络切片中的资源分配问题受到了广泛关注。现有研究大体可分为基于传统优化的方法和基于学习的方法两类。基于优化的方法通常围绕频谱效率、能耗、服务质量或收益最大化等目标，将切片资源分配建模为凸优化、整数规划或启发式搜索问题。此类方法具有较强的理论可解释性，在网络状态变化较缓或约束条件较明确的场景中能够取得较好效果。然而，当网络流量快速波动、用户请求持续变化时，这类方法往往需要频繁重复求解优化问题，计算开销较大，难以满足在线资源管理的实时性要求。

相比之下，基于学习的方法能够通过与环境交互学习资源分配策略，避免在每个时刻显式求解复杂优化问题，因此在动态网络资源管理中表现出更强的适应能力。尤其在RAN切片场景下，切片状态、信道条件和业务需求均具有明显的时变特征，强化学习方法已被广泛用于构建端到端的资源分配策略。然而，大多数现有方法仍主要关注单步即时分配问题，即根据当前观测状态独立地为各切片分配资源，而对未来业务波动和资源持续占用效应考虑不足，这限制了其在复杂动态场景下的应用效果。

### 2.2 强化学习在网络切片中的应用

近年来，深度强化学习已成为解决网络切片资源管理问题的重要技术路线。已有工作采用Q学习、深度Q网络、Actor-Critic以及PPO等方法，对切片间资源分配、切片资源共享和层级资源控制等问题进行了探索。这些方法的共同特点是将网络状态映射为强化学习中的状态空间，将带宽、PRB数量或资源比例映射为动作空间，再通过奖惩机制引导智能体学习资源分配策略。相较于静态优化方法，深度强化学习能够在缺乏精确数学模型的条件下直接从交互数据中逼近长期最优控制策略，因此在非平稳业务环境下具有明显优势。

尽管如此，现有基于深度强化学习的RAN切片资源分配研究仍存在若干不足。首先，许多方法严重依赖当前时刻观测信息，缺乏对短时未来业务趋势的显式建模，因而在流量突发场景中容易出现资源配置滞后。其次，不少研究默认资源动作在当前阶段即时生效并完全释放，较少考虑资源在多个决策阶段之间的持续占用或资源预留问题。再次，已有大量工作将切片数量设定为固定，或采用较理想化的业务模型，这使得策略在更复杂业务波动场景下的适应能力仍有待检验。基于此，在强化学习框架中进一步引入短时流量预测信息，并结合更贴近实际的资源占用机制，具有较高的研究价值。

### 2.3 基于流量预测的切片资源管理方法

随着网络业务时变性增强，利用预测算法辅助资源管理逐渐成为网络切片研究中的重要方向。其中，长短期记忆网络（LSTM）因能够较好地捕捉时间序列中的长期依赖关系，已被广泛用于流量预测、负载预测和信道状态预测等问题。已有工作利用霍尔特-温特斯模型、LSTM或其变体预测未来业务需求，再将预测结果用于切片准入控制、容量规划或资源调度，以提升系统对未来请求变化的响应能力。相较于完全被动的反馈式资源控制，预测辅助方法能够在一定程度上实现前瞻性的资源预留与风险缓解。

然而，现有预测辅助资源分配研究仍存在两个较为突出的问题。其一，预测模块与控制模块的耦合方式往往较为粗糙，一些方法仅将预测结果作为外部参考，而未将其有机融入强化学习状态表示之中。其二，相关研究通常更关注负载变化本身，而较少同步考虑资源在相邻决策阶段中的持续占用效应，因此难以充分体现预测信息在短时资源预留与动态调整中的价值。对于RAN切片场景而言，若能将短时流量预测、切片资源占用信息和强化学习决策统一到同一框架中，有望更好地提升突发业务条件下的资源管理能力。

### 2.4 现有工作的不足与本文定位

综上所述，现有RAN切片资源分配研究虽然已在强化学习和预测辅助资源管理方面取得较多进展，但仍存在以下不足：1）多数方法主要依据当前观测状态做决策，缺乏对未来短时业务波动的显式感知；2）对资源在多个决策阶段间持续占用的问题考虑不足，难以支持前瞻性的资源预留；3）在突发业务与资源紧张并存场景下，对SLA保障与资源开销联合权衡的研究仍不充分。

基于上述分析，本文聚焦固定切片数量场景下的RAN切片动态资源分配问题，研究一种面向突发业务的预测增强型资源分配方法。本文采用LSTM预测短时外生业务量，并将预测结果与实时切片状态、跨step资源占用状态及剩余资源信息联合输入强化学习智能体，以增强策略对未来流量变化的响应能力。同时，本文通过构建跨step PRB占用机制和突发业务切换场景，更贴近实际短时资源管理过程，从而为分析切片SLA保障与资源利用之间的权衡关系提供统一实验框架。

## 3. 系统模型与问题描述

### 3.1 RAN切片场景与业务模型

本文考虑一个单基站RAN网络切片场景。基站侧拥有总计B个可分配物理资源块（Physical Resource Blocks, PRBs），需要在固定数量的K个网络切片之间动态分配无线资源。每个切片具有独立的业务需求和SLA约束。结合当前实现，本文主要考虑eMBB切片与mMTC切片共存的场景，并在实验中采用3个eMBB切片和2个mMTC切片共享总计200个PRB的配置。

系统以离散决策阶段运行。在每个决策阶段中，环境首先对上一阶段内的业务统计量和服务状态进行聚合，再由控制智能体依据当前观测结果输出本阶段的资源分配动作。对于eMBB切片，观测状态主要包括恒定比特率流量、可变比特率流量、吞吐量、已分配PRB、队列长度和信道相关统计量等信息；对于mMTC切片，观测状态主要包括活跃设备数量、重复接入相关统计量和业务时延等信息。设第t个决策阶段的原始切片状态为

x_t = [o_t^{(1)}, o_t^{(2)}, ..., o_t^{(K)}],

其中o_t^{(i)}表示第i个切片在阶段t的局部状态向量。本文中，eMBB切片的局部状态维度为10，mMTC切片的局部状态维度为3。

为增强场景中的业务动态性，本文在基础业务模型上引入突发业务切换机制。具体而言，切片业务在正常态与突发态之间切换，不同状态下对应不同的业务到达率，从而更真实地模拟网络流量在短时间内快速波动的情况。该机制使资源分配问题不再是相对平稳的静态控制问题，而成为需要兼顾短时突发风险的动态决策问题。

### 3.2 跨step PRB占用机制

传统切片资源分配方法通常默认资源动作仅在当前决策阶段内生效，即本阶段分配的资源在下一阶段开始时完全释放。这种设定虽然简化了控制过程，但难以反映实际网络中资源短时预留、持续占用和延续调节的需求。为此，本文在环境中引入跨step PRB占用机制。

具体地，在第t个决策阶段，控制智能体不仅需要决定每个切片在当前阶段立即使用的PRB数量，还需要决定是否为该切片保留一部分能够跨多个阶段持续占用的PRB。设a_t为阶段t的联合动作，则可表示为

a_t = [c_t^{(1)}, ..., c_t^{(K)}, n_t^{(1)}, ..., n_t^{(K)}, ρ_t],

其中c_t^{(i)}表示分配给切片i的跨step占用PRB数量，n_t^{(i)}表示切片i在当前阶段立即使用的PRB数量，ρ_t表示系统层面的保留或未使用资源份额。与之对应，系统需满足可用资源约束，即各切片当前分配、跨阶段占用和保留资源之和不超过当前可支配资源总量。

对于跨step分配的资源，系统将其视为具有有限持续时长的资源租约。租约在后续若干决策阶段内保持有效，并持续占用全局PRB池中的可分配资源。因而，在任一决策阶段，切片i的实际可用PRB不再仅由当前动作中的n_t^{(i)}决定，还与此前尚未到期的跨step占用资源有关。设l_t^{(i)}表示阶段t时切片i仍处于有效期内的占用PRB数量，则切片i在阶段t的有效资源可表示为

u_t^{(i)} = n_t^{(i)} + l_t^{(i)}.

通过该机制，资源分配问题由单步静态动作扩展为具有短时记忆和持续占用特征的动态控制问题。控制智能体需要在当前资源消耗与未来资源可用性之间进行权衡，这也为引入短时业务预测提供了合理动机。

### 3.3 预测增强的状态表示

为使资源分配策略具备前瞻性，本文利用LSTM对外生业务量进行短时预测，并将预测结果纳入强化学习状态空间。预测对象并非奖励或SLA结果本身，而是影响资源需求变化的关键业务量。对于eMBB切片，本文主要预测其恒定比特率流量和可变比特率流量；对于mMTC切片，主要预测下一阶段的新到达设备数量。

设h_t表示阶段t的外生业务特征向量，则有

h_t = [h_t^{(1)}, h_t^{(2)}, ..., h_t^{(K)}],

其中对于eMBB切片，h_t^{(i)}由两类流量分量组成；对于mMTC切片，h_t^{(i)}由新到达设备数量构成。LSTM利用最近H个阶段的历史外生业务序列

[h_{t-H+1}, ..., h_t]

生成对下一阶段业务量的预测结果ĥ_{t+1}。在当前实现中，历史窗口长度设置为10。

在此基础上，本文构建增强后的系统状态为

s_t = [x_t, l_t, b_t^{rem}, ĥ_{t+1}],

其中x_t为当前阶段的原始切片状态，l_t = [l_t^{(1)}, ..., l_t^{(K)}]表示各切片的跨step占用资源状态，b_t^{rem}表示系统当前剩余可分配PRB数量，ĥ_{t+1}表示由LSTM给出的下一阶段外生业务预测。通过该状态表示，强化学习智能体既能够感知当前网络运行状态，也能够利用未来短时业务变化趋势辅助决策。

需要说明的是，本文采用“结构化预测接入”方式，而非将完整时序序列直接交由强化学习网络端到端学习。这样处理主要基于两点考虑：一方面，可以显式区分“外生业务演化”与“资源控制决策”两个过程，增强方法的可解释性；另一方面，单独构建预测模块有助于降低强化学习网络同时承担预测与控制两项任务所带来的训练难度，从而改善样本利用效率和训练稳定性。

### 3.4 优化目标与马尔可夫决策过程建模

本文将上述资源分配问题建模为马尔可夫决策过程（Markov Decision Process, MDP）。在每个决策阶段t，智能体观测到系统状态s_t后，输出联合动作a_t，并与环境交互获得即时奖励r_t以及下一状态s_{t+1}。资源分配策略记为π(a_t|s_t)，其目标是在长期决策过程中尽可能降低切片SLA违约，同时控制PRB占用开销。

从优化目标看，本文关注两个方面：一是在突发业务条件下尽可能降低各类切片的SLA违约风险；二是在满足业务需求的前提下尽可能减少系统资源占用。设v_t表示阶段t的SLA违约程度，cost_t表示阶段t的资源占用开销，则问题可抽象为在长期意义下同时最小化违约与资源成本。对应地，强化学习目标可写为最大化策略的期望累计折扣回报：

max_π E_π [ Σ_{t=0}^{T} γ^t r_t ],

其中γ为折扣因子，r_t为依据SLA满足情况与资源消耗共同构造的即时奖励。为提升训练稳定性，本文并未直接采用“违约即大惩罚、满足即奖励剩余资源”的简单稀疏奖励形式，而是在后续方法设计中进一步构建基于SLA裕量的稠密奖励函数，使智能体能够从更细粒度的反馈中学习资源分配策略。

综上，本文研究的问题可以概括为：在固定切片数量、业务突发性明显且资源可跨阶段持续占用的RAN切片环境中，如何利用短时流量预测结果增强强化学习决策能力，从而实现兼顾SLA保障与资源开销控制的动态PRB分配。

## 4. 预测增强型动态资源分配方法

为解决突发业务场景下传统强化学习资源分配策略前瞻性不足的问题，本文提出一种预测增强型动态资源分配方法。该方法由短时业务预测模块和强化学习资源分配模块两部分组成：前者利用LSTM对下一阶段的关键外生业务量进行预测，后者以预测增强后的状态作为输入，输出切片间PRB的动态分配动作。在此基础上，本文进一步设计跨step资源占用动作机制和兼顾SLA保障与资源成本的奖励函数，使策略能够在满足服务需求的同时对未来短时资源紧张风险进行缓解。

### 4.1 基于LSTM的短时业务预测模块

在RAN切片环境中，流量需求具有明显的时间相关性与突发性，尤其在业务由正常态切换至突发态时，仅依据当前阶段观测量难以及时反映下一阶段资源需求变化。为此，本文使用LSTM构建短时业务预测模块，对影响资源需求的关键外生业务量进行一步预测。

结合当前环境设定，本文并不直接预测完整切片状态，而是仅预测与未来资源需求紧密相关的外生业务分量。对于eMBB切片，预测量包括恒定比特率业务流量和可变比特率业务流量；对于mMTC切片，预测量包括下一阶段新到达设备数量。设第t阶段的外生业务特征向量为h_t，则基于最近H个阶段的历史业务序列

[h_{t-H+1}, h_{t-H+2}, ..., h_t]

LSTM输出下一阶段的预测结果ĥ_{t+1}。在本文实现中，历史窗口长度取H=10。LSTM模型采用单层循环结构，并通过全连接层将隐藏状态映射为与输入维度一致的预测输出。为保持预测模块与资源分配模块之间的量纲一致性，本文对输入业务特征进行归一化处理，并在输出端将预测结果映射回原始业务量尺度。

与直接在线随机初始化训练相比，本文先基于离线收集的业务历史序列对LSTM进行预训练，再将预训练权重接入强化学习训练过程。离线预训练的目的在于使预测模块在强化学习初始阶段即可提供相对稳定的先验信息，避免策略在训练早期受到低质量预测结果的过度干扰。需要指出的是，当前训练框架保留了在线微调接口，但本文主实验采用离线预训练权重直接接入强化学习环境，未启用在线微调机制。

### 4.2 预测增强的状态表示构建

为使强化学习策略能够有效利用短时预测信息，本文将LSTM输出与当前系统状态进行融合，构造预测增强后的联合状态表示。与仅依据当前观测状态的传统方法不同，本文旨在使智能体在做出资源分配决策时，不仅能够刻画“当前系统处于何种状态”，还能够利用“下一阶段可能出现何种业务变化”的先验信息。

具体而言，联合状态由四部分组成：1）当前阶段的原始切片状态；2）各切片当前仍处于有效期内的跨step资源占用状态；3）系统当前剩余可分配PRB数量；4）LSTM给出的下一阶段关键业务量预测结果。设增强后的状态为s_t，则有

s_t = [x_t, l_t, b_t^{rem}, ĥ_{t+1}].

其中，x_t反映当前切片的业务与服务状态，l_t反映由历史动作带来的持续资源占用，b_t^{rem}反映全局资源池当前的剩余能力，而ĥ_{t+1}则提供下一阶段潜在业务压力的先验信息。通过该设计，智能体能够在决策阶段同时考虑当前资源消耗、已有资源承诺和未来短时业务变化趋势，从而为资源控制提供更具前瞻性的状态表征。

本文仍采用“结构化预测接入”而非端到端时序建模的方式。其原因在于：一方面，该方式有助于保持业务演化建模与资源控制建模之间的功能分离，从而增强方法的可解释性；另一方面，在当前样本规模和训练条件下，模块化设计更有利于降低训练难度并提升实验可控性。

### 4.3 面向资源预留的动态动作设计

传统切片资源分配策略通常只输出各切片在当前阶段立即使用的资源数量，这类动作形式更适用于平稳场景下的瞬时调度。然而，在突发业务和短时资源波动条件下，仅依赖单步即时分配容易出现“当前阶段分配可行，而后续阶段调整滞后”的问题。为缓解这一问题，本文在动作设计中引入面向资源预留的联合控制机制。

设第t阶段的动作向量为

a_t = [c_t^{(1)}, ..., c_t^{(K)}, n_t^{(1)}, ..., n_t^{(K)}, ρ_t],

其中c_t^{(i)}表示切片i在当前阶段申请的跨step占用PRB数量，n_t^{(i)}表示切片i当前阶段立即使用的PRB数量，ρ_t表示系统级保留或未使用资源。与单一即时分配不同，该动作机制允许智能体在当前阶段为未来若干阶段预先锁定部分资源，从而为短时业务突发场景下的资源缓冲提供实现途径。

对于跨step动作，系统并不将其简单视为固定时长资源锁定，而是结合切片当前业务状态、即时分配量和历史统计信息，为其估计有限持续时间，并以资源租约的形式记录在系统状态中。这样一来，跨step动作不仅影响当前资源池的可用容量，也会影响后续阶段的剩余资源水平和各切片的实际可用资源数量。智能体因此需要在“增加当前资源投入以降低即时违约风险”和“保留未来资源以缓解后续压力”之间进行权衡。

此外，为匹配连续动作强化学习算法的输出形式，本文在环境交互前对动作进行由比例到整数PRB的映射。具体而言，首先依据动作各分量的相对权重计算当前阶段可实际使用的PRB总量，再分别在跨step和当前step两组动作内部进行整数分配，以保证动作满足资源约束并能够映射到PRB数量层面的实际执行。通过这一设计，强化学习智能体能够在连续动作空间中学习策略，而环境侧则执行满足约束条件的离散资源分配结果。

### 4.4 奖励函数设计

RAN切片资源分配的核心目标是在满足不同切片SLA要求的同时尽可能减少资源开销。因此，奖励函数需要同时反映服务保障水平与资源利用成本。若采用传统稀疏奖励形式，例如“违约则给予固定大惩罚，否则奖励剩余资源”，虽然直观，但反馈过于粗糙，难以为智能体提供稳定的训练信号。尤其在多切片、多约束场景下，策略仅能从“是否违约”的二值信号中学习，容易导致训练震荡和收敛缓慢。

为此，本文构建基于SLA裕量的稠密奖励函数。对于每个切片，首先依据其业务类型和SLA约束，计算各项服务指标相对阈值的满足裕量。对于eMBB切片，本文分别从吞吐量、PRB使用和队列积压等角度评估CBR与VBR业务的满足程度；对于mMTC切片，则主要围绕时延等关键约束构造裕量。随后，将所有切片中的正裕量与负裕量分别聚合，形成系统在当前阶段的满足冗余与违约程度。

在此基础上，本文将即时奖励设计为分段形式：当任一切片发生SLA违约时，奖励由固定屏障惩罚和负裕量惩罚项组成；当所有切片均满足SLA时，奖励由基础正奖励、正裕量激励项和资源开销惩罚项共同组成。该设计的含义在于：当系统发生违约时，智能体应优先学会避免服务失败；当所有切片均满足约束后，策略则需要进一步学习如何在不过度占用资源的前提下维持服务质量，从而实现由“可行”向“高效”的优化。

相较于稀疏奖励，该奖励设计能够为智能体提供更细粒度的反馈。例如，两个同样未违约的状态可以因裕量大小不同而获得不同奖励，从而引导策略偏向更稳健的资源配置；两个同样发生违约的状态也可以因违约程度不同而受到差异化惩罚。这种稠密反馈机制有助于提高强化学习训练的稳定性，并增强策略对多切片资源约束边界的辨识能力。

### 4.5 策略训练流程与实现细节

在强化学习训练阶段，本文首先构造包含预测模块和资源分配模块的交互环境。每个环境实例在初始化后，基站侧根据切片配置、业务模型和信道模型生成相应切片状态；随后，在每个决策阶段，环境先更新历史资源租约状态并释放已到期资源，再根据当前状态和历史业务序列生成预测增强状态，供智能体进行决策。智能体输出联合动作后，环境将动作映射为整数PRB分配方案，完成跨step占用记录与当前step资源分配，并据此推进一个完整观测周期。

考虑到资源分配问题具有较强的时序依赖和训练不稳定性，本文在训练时采用分阶段训练策略。第一阶段主要强调策略对基础可行行为的学习；第二阶段逐步增强对资源效率的约束；第三阶段进一步强化对复杂业务波动场景下长期收益的优化。通过分阶段调整训练参数和奖励相关系数，策略能够由“先满足SLA”逐步过渡到“在满足SLA的前提下降低资源成本”，从而提升训练过程的稳定性。

在实现上，本文采用基于深度强化学习的连续动作算法作为资源分配策略求解器，并使用并行环境进行采样以提高训练效率。预测模块使用预训练得到的LSTM参数进行初始化；环境侧记录每个决策阶段的SLA违约数、奖励、资源占用、分配动作以及外生业务序列，用于后续实验分析与补充消融研究。通过上述流程，本文形成了一个包含“业务预测—状态增强—联合动作输出—资源租约更新—奖励反馈”的完整闭环，为后续实验验证提供了实现基础。

## 5. 实验设置与结果分析

### 5.1 仿真环境与参数设置

本文基于已有RAN网络切片仿真环境开展实验，并在其基础上引入突发业务切换、跨step PRB占用以及预测增强状态表示等机制。实验重点采用固定切片数量场景，即3个eMBB切片与2个mMTC切片共享200个PRB资源。每个决策阶段对应50个slot的观测窗口，系统在该窗口内聚合业务和服务信息后进行一次资源分配决策。

在业务建模方面，eMBB切片同时包含CBR和VBR两类业务，二者的到达率均在正常态与突发态之间切换，以模拟复杂业务波动场景。其中，CBR业务在正常态与突发态下的平均到达率分别为2/60和8/60，VBR业务在正常态与突发态下的平均到达率分别为5/60和25/60。mMTC业务采用设备到达与接入时延相关建模。系统总惩罚系数设置为1000，eMBB与mMTC切片分别具有对应的吞吐、队列和时延等SLA约束。

在训练设置方面，本文采用4个并行环境进行采样，训练分为三个阶段，每个阶段训练步数均为102400，总训练步数为307200；评估阶段长度为2560步。预测模块的历史输入窗口长度设置为10，隐藏层维度设置为128，并使用离线预训练权重进行初始化。在当前主实验设置下，LSTM模块以离线预训练模式接入强化学习环境，不启用在线微调。实验过程中，环境会记录每个决策阶段的奖励、资源占用、违约数和外生业务序列，用于后续性能分析。

### 5.2 对比算法与评价指标

为评估所提框架的性能，本文选取多种常见深度强化学习算法作为对比基线，包括PPO、A2C、DDPG和TRPO。同时，实验也保留了代码框架中的RaRPPO结果，以及接入LSTM预测模块后的LSTMDDPG结果，用于比较不同策略配置在统一环境下的表现。需要说明的是，本文当前实验的重点在于验证预测增强资源分配框架在现有环境中的可行性，因此实验结果主要用于考察不同策略在统一环境下对SLA保障与资源占用之间权衡关系的表现，而不将其简单理解为对所有现有算法的全面超越。

本文采用以下指标进行评价：

1）平均奖励，用于衡量策略在长期累计目标下的综合表现；

2）平均违约数，用于反映各决策阶段内切片SLA违约程度；

3）违约阶段占比，用于描述在评估窗口内发生违约的阶段比例；

4）平均资源占用，用于衡量系统在评估期间的PRB消耗水平。

其中，平均违约数和违约阶段占比主要体现SLA保障能力，平均资源占用反映资源成本，而平均奖励则综合体现二者在奖励函数下的平衡结果。

### 5.3 整体性能对比

本文首先在固定切片数量、突发业务场景下，对不同算法在评估窗口内的整体性能进行比较。表1给出了各方法在最后2560步评估阶段中的主要结果。由于实验采用4个并行环境，最终结果由4个环境的评估样本共同统计得到，总评估样本数为10240。

| TRPO | 18.8605 | 0.1575 | 0.1315 | 194.9546 |

从表1可以看出，在当前实验设置下，不同算法在SLA保障与资源占用方面呈现出不同的权衡特征。PPO在平均奖励、平均违约数和违约阶段占比等指标上表现相对较优，DDPG次之，A2C居中，而TRPO在当前场景下表现相对较弱。与无预测的DDPG相比，LSTMDDPG在平均资源占用上略有下降，但其平均违约数和违约阶段占比高于DDPG。这表明，在当前参数配置和训练设置下，引入预测模块尚未直接转化为更优的综合控制效果。

上述结果说明，预测增强机制在现阶段已经成功接入资源分配闭环，并对策略行为产生影响，但其有效性仍受到预测质量、奖励设计、动作机制以及策略优化器适配程度的共同影响。因此，本文更倾向于将当前结果理解为对“预测增强资源分配框架可行性”的初步验证，而非对“预测增强一定优于无预测策略”的直接证明。

### 5.4 结果分析与讨论

结合当前实验结果，可以从以下几个方面对现象进行分析。

首先，预测信息的引入并不会自动带来性能增益。本文采用的是“预测结果拼接到状态空间”的方式，即预测模块仅为强化学习策略提供额外先验信息，而最终能否从该信息中学习出更优策略，仍取决于强化学习算法本身的表达能力与训练效率。如果预测模块提供的信息未能被控制策略充分利用，或者预测误差在训练过程中被放大，则可能出现“接入预测但综合效果未提升”的情况。

其次，跨step PRB占用机制增强了动作空间的表达能力，但也提高了策略学习难度。与单步即时分配相比，当前动作不仅影响本阶段的资源使用，还会通过租约机制影响后续阶段的剩余资源水平。这种短时资源记忆机制虽然更贴近动态资源管理过程，但同时也使策略优化问题具有更强的时序耦合性，从而增加了训练难度。

再次，当前实验采用离线预训练LSTM而未启用在线微调。该设定有利于保持训练初期的稳定性，但也意味着预测模块尚未根据当前策略交互过程中形成的数据分布进行进一步适应。因此，预测模型与控制环境之间仍可能存在分布偏差，进而影响预测信息的有效利用。

最后，从整体结果看，本文当前工作更适合被理解为一个面向突发业务场景的预测增强型资源分配框架，而不是一个已经在所有基线上全面领先的成熟最优方案。当前结果揭示了预测信息、持续资源占用机制与强化学习控制之间存在值得进一步研究的耦合关系，也说明后续仍需通过消融实验和参数调优进一步分析不同模块对最终性能的实际贡献。

### 5.5 进一步验证方案

为进一步验证本文方法中各模块的实际作用，后续实验可重点围绕以下几个方面展开：

1）有无LSTM预测的对比实验，用于分析预测增强状态表示对策略性能的影响；

2）有无跨step占用机制的对比实验，用于分析资源租约机制对动态资源管理能力的作用；

3）稠密奖励与原始稀疏奖励的消融实验，用于分析奖励设计对训练稳定性和最终性能的影响；

4）平稳流量场景与突发流量场景的对比实验，用于说明本文问题设定相较于传统平稳场景的挑战性。

若条件允许，进一步引入Oracle预测作为上界参考，将有助于更清晰地判断当前方法的性能瓶颈究竟来自预测精度不足，还是来自控制策略对预测信息利用不充分。

## 6. 结论与展望

本文围绕固定切片数量场景下的RAN网络切片动态资源分配问题，研究了一种面向突发业务的预测增强型资源分配方法。针对传统强化学习方法主要依赖当前观测状态、缺乏对未来短时业务波动显式感知的问题，本文将LSTM短时业务预测模块与强化学习资源分配策略相结合，并在现有切片环境基础上进一步引入跨step PRB占用机制、突发业务切换机制以及稠密SLA奖励设计，从而构建了一个包含“业务预测—状态增强—动态动作—奖励反馈”的完整资源分配闭环。

从方法层面看，本文完成了三个方面的工作：一是利用LSTM对eMBB业务流量和mMTC新到达设备数量进行短时预测，并将预测结果作为额外先验信息接入强化学习状态空间；二是将资源动作由单步即时分配扩展为“跨step占用分配 + 当前step分配”的联合控制形式，使资源策略能够显式考虑短时资源持续占用过程；三是设计了兼顾SLA违约程度与资源开销的稠密奖励机制，为强化学习训练提供更细粒度的反馈信号。

从当前实验结果看，本文方法已经在仿真环境中完成了有效实现，并验证了预测增强状态表示与持续资源占用机制能够被纳入统一控制框架之中。在固定切片数量和突发业务场景下，不同强化学习算法在SLA保障和资源占用之间呈现出不同的权衡关系，说明本文所构建的实验环境和评价框架能够用于分析预测信息、动作设计和奖励机制对切片资源分配行为的影响。同时，当前结果也表明，预测模块的引入尚未在现有参数配置下直接转化为显著优于无预测基线的综合性能，这说明预测增强机制的有效性仍依赖于预测质量、策略优化器适配程度以及不同模块之间的协同设计。

总体而言，本文的主要价值在于提出并实现了一种面向突发业务场景的预测增强型RAN切片资源分配框架，为后续深入分析“预测信息如何改善资源控制”提供了基础。与将预测仅作为外部参考的方式不同，本文通过状态增强和动作扩展，将短时业务预测、持续资源占用和强化学习控制更紧密地结合起来，从而为研究复杂动态业务条件下的智能化切片资源管理提供了一条可行路径。

后续工作可从以下几个方向继续推进。首先，需要通过有无预测、有无跨step占用以及稠密奖励与稀疏奖励之间的消融实验，更系统地分析各模块对最终性能的实际贡献。其次，可以引入Oracle预测作为上界参考，并结合在线微调机制进一步提升预测模块对当前环境分布的适应能力。再次，当前研究仅聚焦于固定切片数量场景，后续可进一步扩展到切片数量动态变化的场景，并探索更适合变长状态表示的策略结构。最后，还可以继续向切片内用户级调度问题延伸，构建由切片间资源分配到切片内资源调度的层级化控制框架，以提升方法在更复杂实际网络环境中的适用性。
